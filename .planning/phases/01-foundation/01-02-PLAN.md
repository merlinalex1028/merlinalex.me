## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `<head>` pre-paint script → DOM | Inline `<script is:inline>` runs synchronously before any DOM paints. Wrapped in `try/catch` to survive private-mode browsers / SSR bots. No third-party content. |
| `window.__atmo__.set()` → `localStorage` | Only the pre-paint script + the two user-input handlers (ThemeSwitcher click, IntensityBadge click) write to localStorage. No other code path mutates theme/atmo state. |
| Inline scripts in components → DOM | ThemeSwitcher, IntensityBadge, NoticeBar each ship their own `<script is:inline>` IIFE. All read from `window.__atmo__` (the pre-paint contract) — they do not parse or trust any external input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in plan 01-02 (all deps came from plan 01-01's scaffold) |
| T-02-01 | Tampering | Pre-paint script | mitigate | The IIFE is wrapped in `try/catch`; on failure, the script sets `data-theme="light"` and `data-atmo="full"` as safe defaults rather than throwing. The `set()` method validates that `patch.theme` / `patch.atmo` are non-empty strings before writing to localStorage (defensive — localStorage values are coerced to strings, but nullish writes are a no-op). |
| T-02-02 | Spoofing | `window.__atmo__` consumer | mitigate | The pre-paint script is the SOLE owner of `window.__atmo__`. The `set()` method's listener iteration wraps each callback in `try { fn(this) } catch (e) { /* ignore */ }` so a misbehaving Phase 5 listener cannot break the pre-paint contract. |
| T-02-03 | Information Disclosure | inline scripts | accept | The inline scripts contain no secrets, no PII, no tokens. They reference CSS class names and DOM selectors only. |
| T-02-04 | Repudiation | Theme/atmo state | mitigate | `set()` writes both to `localStorage` AND to the `data-*` attribute in a single call, so a misbehaving consumer cannot observe a state where they disagree. Listeners are notified after both are written. |
| T-02-05 | Denial of Service | Notice dismiss localStorage | accept | `localStorage["notice:dismissed:<id>"]` is the only notice-related state; size is 1 byte per notice; no DoS surface |
</security>
