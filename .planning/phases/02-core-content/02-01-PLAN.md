## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| author→content | Article markdown is author-controlled (trusted) |
| URL→page | Tag filter via `?tag=` query param processed at build time (no server-side injection vector) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Tag filter URL param | mitigate | Build-time static generation via `getStaticPaths` — no runtime query param processing; tag values are validated against known article tags at build time |
| T-02-02 | Tampering | Article content (markdown) | accept | Author-controlled content, no user input; Shiki handles code highlighting safely at build time |
| T-02-03 | Tampering | medium-zoom library | accept | Mature library (8+ years), loaded from npm, no user input to zoom API |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | `@astrojs/rss`, `medium-zoom`, `markdown-it`, `sanitize-html` all verified in RESEARCH.md package legitimacy audit — all APPROVED |
</threat_model>
