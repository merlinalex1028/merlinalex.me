# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind v4 CSS-first config via `@tailwindcss/vite`) |
| Preset | not applicable |
| Component library | none (custom Astro components) |
| Icon library | inline SVG — consistent with Phase 1 emoji pattern; no dependency |
| Font | Zen Maru Gothic, Noto Sans SC, PingFang SC, Hiragino Sans, system-ui (declared in `src/styles/global.css`) |

**Rationale:** Phase 1 established a zero-dependency icon pattern using emoji characters (ThemeSwitcher, IntensityBadge). Phase 2 extends this with inline SVGs for the handful of semantic icons needed (RSS, share, clipboard, arrow-up, chevron). This avoids adding an icon library dependency for ~8 icons.

---

## Spacing Scale

Declared values (multiples of 4). Uses CSS custom property fallback pattern from Phase 1 (`var(--space-md, 1rem)`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline pill padding, tag chip inner gap |
| sm | 8px | Compact element spacing, list item gaps |
| md | 16px | Default element spacing, card padding, section inner padding |
| lg | 24px | Section padding, card group gaps, article body margins |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks (between TOC and content, before comments) |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions: 56px header height (already established in Phase 1, sticky z-50). 44px minimum touch target for tag chips on mobile (accessibility).

- **Active chip:** `background: var(--color-accent-subtle)`, `color: var(--color-accent)`, `border: 1px solid var(--color-accent)`
- **Hover:** `background: var(--color-surface-muted)`

- **Container:** `padding: 0 1rem`, sticky below header on scroll (`position: sticky`, `top: 56px`, `z-index: 40`, `background: var(--color-bg)`)

### ArticleListItem

- **Layout:** Single row, `display: flex`, `gap: 1rem`, `padding: 1rem`, `border-radius: 20px`

- **Border:** `1px solid var(--color-border)`

## Interaction Contracts

### Tag Filtering (Articles Index)

- **Mechanism:** URL query parameter `?tag={value}` — no client-side JS state
- **"全部" chip:** Links to `/articles` (no query param) — always visible
- **Tag chips:** Link to `/articles?tag={tag}` — selected chip highlighted
- **Page reload:** Full page reload on chip click (static site, no SPA)
- **Scroll position:** Reset to top on filter change (browser default)
- **URL sync:** Browser back/forward navigates between tag filters

### TOC Scroll Highlighting

- **Observer:** `IntersectionObserver` on all h2 and h3 elements in article
- **Config:** `rootMargin: '-80px 0px -80px 0px'` (account for sticky header), `threshold: 0`
- **Behavior:** When heading enters observation zone, add `active` class to corresponding TOC item
- **Multiple visible:** Last heading that entered the zone takes precedence
- **Smooth scroll:** TOC click uses `element.scrollIntoView({ behavior: 'smooth', block: 'start' })`

### Copy Code

- **API:** `navigator.clipboard.writeText(codeText)`
- **Fallback:** `document.execCommand('copy')` with temporary textarea
- **Feedback:** Button text changes to "已复制!", icon changes to checkmark
- **Timeout:** Revert to "复制" after 2000ms
- **Multiple instances:** Each code block gets its own button; state is independent

### Image Lightbox (medium-zoom)

- **Init:** On page load and after `astro:after-swap` (View Transitions)
- **Selector:** `article img` (all images in article prose)
- **Keyboard:** Escape to dismiss, Tab to navigate (medium-zoom handles)
- **Scroll lock:** medium-zoom handles body scroll lock during zoom

### Scroll-to-Top

- **Visibility:** Show when `window.scrollY > window.innerHeight`, hide otherwise
- **Animation:** CSS transition on opacity (150ms)
- **Action:** `window.scrollTo({ top: 0, behavior: 'smooth' })`
- **Throttle:** Scroll listener throttled via `requestAnimationFrame`

### Twikoo Comments

- **Load:** CDN script injected on `client:visible` — only loads when comment section scrolls into view
- **Init:** `twikoo.init({ envId, el: '#tcomment' })`
- **Anonymous:** No login prompt — nickname + optional email/URL fields
- **Admin identification:** Email match in Twikoo config (owner's email)
- **Sticker panel:** Bilibili emote set, triggered by emote button in Twikoo input area
- **Theme sync:** Twikoo reads CSS custom variables; re-init not needed on theme switch (CSS variables update automatically)
