  <action>
Create the Projects listing page and GitHub stars integration. Per D-02 (locked decision):

1. **src/scripts/fetch-github-stars.ts** — Build-time script that:
   - Uses Node.js `fs` to read all `.md`/`.mdx` files from `src/content/projects/`
   - Parses frontmatter using `gray-matter` (already installed as astro dependency) to extract `github` field
   - Converts `https://github.com/owner/repo` URLs to `owner/repo` slugs
   - Fetches `https://api.github.com/repos/{owner}/{repo}` for each with `Accept: application/vnd.github.v3+json` header and `User-Agent: merlinalex.me-build`
   - Uses `GITHUB_TOKEN` env var if available (optional — unauthenticated works for <60 repos)
   - Writes result as `{ "owner/repo": 123, ... }` to `src/data/github-stars.json`
   - On any failure (network, rate limit, missing file), writes `{}` and logs warning — never throws
   - Skips fetch if `src/data/github-stars.json` exists and was modified within the last 24 hours

2. **src/components/works/ProjectCard.astro** — Rich card component. Props: `name: string`, `description?: string`, `url: string`, `github?: string`, `tags: string[]`, `cover?: string`, `stars?: number`. Renders:
   - Cover image (if provided) at top with `width="400" height="200" loading="lazy" alt={name}`
   - Project name as `<h3>` linked to `url` with `target="_blank" rel="noopener noreferrer"`
   - Description paragraph (if provided)
   - Tech tags as horizontal chips — reuse the same CSS pattern as TagChips (`background: var(--color-surface); border: 1px solid var(--color-border); border-radius: 9999px; padding: 2px 10px; font-size: 13px`)
   - GitHub badge: `<a href={github}>` with `<Icon name="mdi:github" />` + stars count (or "—" if undefined). Clickable link to repo.
   - Card styling: same as ArticleListItem (`background: var(--color-surface); border: 1px solid var(--color-border); border-radius: 20px; padding: 1rem;`) with hover `border-color: var(--color-accent)` + `box-shadow: 0 4px 16px rgba(255, 107, 157, 0.12)`

3. **src/pages/works/projects/index.astro** — Page using BaseLayout with Breadcrumb (`[Works, /works]` → `Projects`):
   - Import `getCollection('projects')` and `github-stars.json`
   - Sort: `featured: true` first, then alphabetical by name (projects schema has no `publishedAt`)
   - For each project, look up stars from imported JSON using github URL slug
   - Render grid: `display: grid; grid-template-columns: 1fr; gap: 1rem;` with `@media (min-width: 640px) { grid-template-columns: repeat(2, 1fr); }` and `@media (min-width: 1024px) { grid-template-columns: repeat(3, 1fr); }`
   - Empty state: "还没有项目" message when no projects exist

4. **src/data/github-stars.json** — Create empty `{}` as initial value. The fetch-github-stars script will populate it on build.
