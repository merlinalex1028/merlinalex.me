    - Export `BANGUMI_TYPES` constant: `{ anime: 2, book: 1, music: 3 }` as const
    - Export `async function fetchBangumiData(username: string, subjectType: number): Promise<BangumiItem[]>`:
      - Paginate with limit=50, offset incrementing
      - Fetch `https://api.bgm.tv/v0/users/${username}/collections?subject_type=${subjectType}&limit=${limit}&offset=${offset}`
      - Headers: `{ 'User-Agent': 'merlinalex.me/1.0 (Astro build)' }`
      - Flatten `data.data` array, map each item to BangumiItem shape (extract from nested `subject` object)
      - Break when `allItems.length >= data.total` or response not ok
      - Wrap in try/catch, return empty array on error
    - Export `function readCache(filePath: string): { data: BangumiItem[]; isStale: boolean }`:
      - Read file with `fs.readFileSync`, parse JSON
      - Check `fs.statSync(filePath).mtimeMs` against `Date.now() - 12 * 60 * 60 * 1000`
      - Return `{ data: parsed, isStale }` or `{ data: [], isStale: true }` on error
    - Export `function mergeOverrides(items: BangumiItem[], overrides: BangumiItem[]): BangumiItem[]`:
      - Create Map from items by subjectId
      - For each override, set/replace in map
      - Return array from map values
    - Export `function writeCollectionFiles(data: { anime: BangumiItem[]; books: BangumiItem[]; music: BangumiItem[] }): void`:
      - Write `src/content/anime/list.json` with anime array
      - Write `src/content/books/list.json` with books array
      - Write `src/content/music/list.json` with music array
      - Create directories if they don't exist (`fs.mkdirSync` with `recursive: true`)
      - This bridges the gap between fetch utility and content collection `file()` loaders

    - If no username, log warning and write empty arrays to content files (graceful fallback)

      - If fresh, skip fetch (use existing data)
      - If stale or missing, fetch from Bangumi API
      - Merge with overrides from `src/data/bangumi-override.json`
    - Call `writeCollectionFiles({ anime, books, music })` to write all three content collection files
