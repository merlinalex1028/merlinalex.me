  <name>Task 1: Install tsParticles + create sakura petal asset + PetalsIsland</name>
  <files>package.json, public/petals/sakura.png, src/components/atmosphere/PetalsIsland.astro</files>
  <action>
Install the tsParticles Astro integration and supporting packages:
```
pnpm add @tsparticles/astro @tsparticles/slim @tsparticles/shape-image
```

Create `public/petals/sakura.png` — a small sakura petal image (32x32 PNG). Generate a minimal sakura petal shape using a canvas script or SVG-to-PNG conversion. The petal should be a simple 5-petal cherry blossom silhouette in pink (#FFB7C5) with transparency. If you cannot generate a PNG programmatically, create `public/petals/sakura.svg` instead and reference it in the tsParticles config (tsParticles supports SVG shapes via the `image` shape type).

Create `src/components/atmosphere/PetalsIsland.astro` — an Astro island that renders falling cherry blossom petals using tsParticles.

The component must:
1. Import `isOff`, `isReducedMotion`, `isMobile`, `onAtmoChange` from `../../lib/atmosphere`
2. Render a `<div class="atmo-petals">` container (this class is already gated in `global.css` via `data-atmo="off"`)
3. Use `@tsparticles/astro`'s `<Particles>` component with `client:idle` directive
4. Configure the particles with these exact settings (per D-02 and ATM-02):
   - `fullScreen: false` (contained within the atmo-petals div, not fullscreen)
   - `fpsLimit: 30` (30 FPS throttle per ATM-02)
   - Particle count: `isMobile() ? 15 : 30` (mobile cap per ATM-02: "capped to 20 on mobile" — using 15 for better perf)
   - Color: theme-linked — light mode `['#FFB7C5', '#FF69B4', '#FFC0CB']`, dark mode `['#E84393', '#C0392B', '#FFB7C5']` — read `window.__atmo__?.theme` to pick palette
   - Shape: `type: 'image'`, `image: { src: '/petals/sakura.png', width: 32, height: 32 }`
   - Opacity: `{ value: { min: 0.3, max: 0.8 }, animation: { enable: true, speed: 0.5, minimumValue: 0.1 } }`
   - Size: `{ value: { min: 8, max: 16 }, animation: { enable: true, speed: 2, minimumValue: 4 } }`
   - Move: `direction: 'bottom-right'`, `speed: { min: 1, max: 3 }`, `straight: false`, `outModes: { default: 'out' }`, `gravity: { enable: true, acceleration: 0.5 }`
   - Rotate: `{ value: { min: 0, max: 360 }, animation: { enable: true, speed: 5 } }`
   - Wobble: `{ enable: true, distance: 10, speed: 3 }`
   - `pauseOnBlur: true` (pauses on document.hidden per ATM-02)
   - `detectRetina: true`
5. Gate on intensity: in the script, check `isOff()` or `isReducedMotion()` before initializing. If off/reduced, skip particle engine init entirely and hide the container.
6. Subscribe to intensity changes via `onAtmoChange`: when level changes to 'off' or reducedMotion becomes true, destroy/hide the particle canvas; when level changes to 'subtle' or 'full', re-initialize.
7. The container div must have `pointer-events: none` so the canvas doesn't block page scroll (Pitfall 2 from RESEARCH).

**Important**: The `<Particles>` component from `@tsparticles/astro` handles the `initParticlesEngine` call internally. You do NOT need to call `initParticlesEngine` manually. Just import and use the `<Particles client:idle>` component with the options object.

**Important**: Since the particle count depends on `isMobile()` (a client-side check), and the `<Particles>` component renders server-side first, use a two-phase approach: render the `<Particles>` component with desktop defaults (count: 30), then in the client-side script, override the count to 15 on mobile via the particles API if `isMobile()` returns true. Alternatively, render two `<Particles>` instances conditionally — but the simpler approach is to use the `responsive` option in tsParticles config:
```
responsive: [
  { maxWidth: 768, options: { particles: { number: { value: 15 } } } }
]
```
This is the cleanest approach — tsParticles handles the responsive breakpoint internally.

<task type="auto">
  <name>Task 2: Create CursorTrailIsland + wire both into BaseLayout</name>
  <files>src/components/atmosphere/CursorTrailIsland.astro, src/layouts/BaseLayout.astro</files>
  <action>
Create `src/components/atmosphere/CursorTrailIsland.astro` — a desktop-only cursor trail effect that spawns small petal particles following the mouse.

The component must:
1. Import `isOff`, `isReducedMotion`, `isMobile`, `onAtmoChange` from `../../lib/atmosphere`
2. Render a `<div class="atmo-cursor-trail">` container (class already gated in `global.css`)
3. In a `<script>` block:
   - Guard: if `isOff()`, `isReducedMotion()`, or `isMobile()`, return early and hide the container
   - Add a `mousemove` listener on `document` that spawns a small petal element at the cursor position every ~100ms (throttled)
   - Each petal is a tiny `<span>` with a random rotation, a pink color from the theme palette, and a CSS animation that fades out + falls down over 1 second, then removes itself from the DOM
   - Use `requestAnimationFrame` for the animation, but throttle spawn rate to every 100ms (not every mousemove event)
   - Max 20 active trail particles at once (remove oldest if exceeded)
   - Subscribe to `onAtmoChange`: when level changes to 'off' or reducedMotion becomes true, remove the listener and clear all active particles; when level changes back, re-attach the listener
   - On cleanup (if the page navigates via View Transitions), remove the mousemove listener to prevent leaks

The cursor trail is intentionally lightweight — no tsParticles dependency, just vanilla DOM manipulation with CSS animations. This keeps the bundle small and avoids a second tsParticles instance.

Style the trail particles with:
- `position: fixed; pointer-events: none; z-index: 9999;`
- Size: 8-12px (random)
- Color: `var(--color-accent)` (theme-linked)
- Animation: `@keyframes cursor-fall { 0% { opacity: 1; transform: translateY(0) rotate(0deg); } 100% { opacity: 0; transform: translateY(40px) rotate(180deg); } }` — duration 1s, ease-out

Add the `@keyframes cursor-fall` animation to the component's `<style>` block (scoped).

Modify `src/layouts/BaseLayout.astro`:
- Import both `PetalsIsland` and `CursorTrailIsland` from `'../components/atmosphere/'`

  ```
  </AtmosphereLayer>
  ```
