<task type="auto">
  <name>Task 1: Create BGMIsland with APlayer + MetingJS (muted default + unmute button)</name>
  <files>src/components/atmosphere/BGMIsland.astro</files>
  <action>
Create `src/components/atmosphere/BGMIsland.astro` — an Astro island that loads APlayer + MetingJS from CDN and provides a site-wide BGM player.

The component must:
1. Import `isOff`, `onAtmoChange` from `../../lib/atmosphere`
2. Render the following HTML structure:
   ```html
   <div id="bgm-root" class="atmo-bgm" transition:persist="bgm-player">
     <button id="bgm-unmute" class="bgm-unmute-btn" aria-label="打开音乐" type="button">
       <span class="bgm-unmute-icon">🔇</span>
       <span class="bgm-unmute-text">打开音乐</span>
     </button>
     <div id="aplayer-container"></div>
   </div>
   ```
3. The `transition:persist="bgm-player"` attribute ensures Astro View Transitions preserves this element across page navigation (the APlayer instance stays alive).
4. Style the component:
   - `.atmo-bgm`: `position: fixed; bottom: 20px; left: 20px; z-index: 100;`
   - `.bgm-unmute-btn`: kawaii-styled button with `var(--color-surface)` background, `var(--color-border)` border, `border-radius: 999px`, `padding: 0.5rem 1rem`, `cursor: pointer`, `font-size: 0.9rem`, `color: var(--color-fg)`, `display: flex; align-items: center; gap: 0.4rem`
   - `.bgm-unmute-btn:hover`: `background: var(--color-surface-muted)`
   - Hide `.atmo-bgm` when `data-atmo="off"` via the existing CSS gate in global.css (already covers `.atmo-bgm` pattern if you add the class, OR add `:root[data-atmo="off"] .atmo-bgm { display: none; }` to global.css)
   - When APlayer is loaded, hide the unmute button and let APlayer's own UI handle play/pause
   - APlayer container should be scoped with `all: initial` on the wrapper to prevent Tailwind CSS conflicts (Pitfall 6 from RESEARCH)

5. In the `<script>` block:
   - Guard: if `isOff()`, hide the root element and return early
   - Check `localStorage.getItem('bgm:unmuted')` — if `'true'`, skip the unmute button and load APlayer immediately
   - On unmute button click:
     a. iOS AudioContext.resume(): `if (window.AudioContext || window.webkitAudioContext) { new (window.AudioContext || window.webkitAudioContext)().resume(); }`
     b. Call `loadAPlayer()`
     c. Set `localStorage.setItem('bgm:unmuted', 'true')`
     d. Hide the unmute button
   - `loadAPlayer()` function:
     a. Create a `<link>` element for APlayer CSS: `https://cdn.jsdelivr.net/npm/aplayer@1.10.1/dist/APlayer.min.css`
     b. Create a `<script>` element for APlayer JS: `https://cdn.jsdelivr.net/npm/aplayer@1.10.1/dist/APlayer.min.js`
     c. On APlayer script load, create a `<script>` element for MetingJS: `https://cdn.jsdelivr.net/npm/meting@2.0.2/dist/Meting.min.js`
     d. On MetingJS load, create a `<meting-js>` custom element with attributes:
        - `server="netease"` (NetEase Cloud Music)
        - `type="playlist"`
        - `id="2884035"` (placeholder playlist ID — a public NetEase playlist of lofi/anime music; the user can change this later)
        - `autoplay="false"`
        - `mutex="true"` (only one audio plays at a time)
        - `preload="none"` (don't preload audio data)
        - `theme="var(--color-accent)"` (theme-linked player accent)
     e. Append the `<meting-js>` element to `#aplayer-container`
   - Subscribe to `onAtmoChange`: when level changes to 'off', hide the root; when level changes back, show it. Do NOT destroy the APlayer instance — just hide/show the container so playback state is preserved.
   - All CDN script loads must use `document.head.appendChild(script)` pattern (dynamic injection), NOT `<script src="...">` in the template (these need to load after the unmute click, not on page load).

6. The APlayer CSS needs isolation from Tailwind. Add to the component's `<style>` block:
   ```css
   #aplayer-container {
     all: initial;
   }
   #aplayer-container .aplayer {
     /* Override only what's needed to match site design */
   }
   ```
   This `all: initial` reset prevents Tailwind's preflight from breaking APlayer's UI.
  </action>

    - `localStorage['bgm:unmuted']` persists the unmuted state across page loads
    - APlayer CSS isolated from Tailwind via `all: initial`
    - `pnpm build` exits 0
  </done>

Modify `astro.config.mjs` to enable View Transitions. This is required for `transition:persist` on the BGM player to work across page navigation (Pitfall 4 from RESEARCH).

Add `viewTransitions: true` to the Astro config's top level (NOT inside `integrations`). The config currently looks like:
```js
export default defineConfig({
  site: 'https://merlinalex.me',
  integrations: [icon(), mdx(), sitemap(), pagefind()],
  vite: { plugins: [tailwindcss(), yaml()] },
  trailingSlash: 'never',
  build: { inlineStylesheets: 'auto' },
});
```

Add `viewTransitions: true` after the `build` block. This enables Astro's View Transitions API globally — all `<a>` tags will use client-side navigation by default, preserving DOM elements marked with `transition:persist`.

**Note**: View Transitions in Astro 6 are enabled via the `viewTransitions` boolean in config, NOT by adding a `<ViewTransitions />` component to the layout (that was the Astro 4 pattern). In Astro 6, setting `viewTransitions: true` in config automatically injects the transition script.
