## Adding Models to Live2DIsland
Edit `src/components/atmosphere/Live2DIsland.astro` and update the `model` array in `createWidget()`:
```js
model: [
  { path: '/models/your-model/model.json' },
]
```
```

Create `src/components/atmosphere/Live2DIsland.astro` — an Astro island that renders the Live2D mascot.

The component must:
1. Import `isOff`, `onAtmoChange` from `../../lib/atmosphere`
2. Render:
   ```html
   <div class="atmo-live2d" id="l2d-root">
     <img
       class="l2d-fallback"
       src="/models/fallback.svg"
       alt="站点吉祥物"
       loading="lazy"
       width="200"
       height="300"
     />
   </div>
   ```
3. Style:
   - `.atmo-live2d`: `position: fixed; bottom: 0; right: 20px; z-index: 100; width: 200px; height: 300px; pointer-events: auto;`
   - `.l2d-fallback`: `width: 100%; height: auto; transition: opacity var(--duration-base) var(--easing-default);`
   - The `.atmo-live2d` class is already gated in `global.css` via `data-atmo="off"`

4. In the `<script>` block (per D-01):
   - Guard: if `isOff()`, hide the root and return early
   - Try to load Live2D using `createWidget` from `l2d-widget`:
     ```js
     import { createWidget } from 'l2d-widget';
     ```
   - Pass model configuration:
     ```js
     const widget = createWidget({
       model: [
         { path: '/models/cat-black/model.json' },
         { path: '/models/cat-white/model.json' },
       ],
       tips: {
         welcome: '欢迎回来~',
         touch: '别戳我啦~',
       },
     });
     ```
   - **Try-catch the entire createWidget call** (per D-01: "try-load-all-devices with static PNG fallback"). If it throws (WebGL not supported, OOM, Cubism SDK load failure), show the fallback SVG and return.
   - On success: hide the fallback image (`fallback.style.display = 'none'`), the Live2D canvas takes over
   - On failure: keep the fallback visible, log a warning to console
   - Subscribe to `onAtmoChange`: when level changes to 'off', call `widget.sleep()` if available; when level changes back, let the widget auto-wake on interaction
   - Model switcher: the `createWidget` call with an array of models enables the built-in model switcher in l2d-widget. Users click the mascot to cycle models.

5. **Important**: l2d-widget handles Cubism SDK loading internally. You do NOT need to manually load the Cubism SDK. The `createWidget()` function auto-detects Cubism 2 vs Cubism 6 and loads the appropriate runtime.

6. **Model files**: Since this is an MVP with no actual Live2D models yet, the component should gracefully handle missing model files. If `createWidget()` fails because `/models/cat-black/model.json` returns 404, the catch block shows the fallback SVG. The user adds real model files later.
