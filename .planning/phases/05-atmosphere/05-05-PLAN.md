Create `src/components/atmosphere/ContextMenuIsland.astro` — an Astro island that provides a custom context menu triggered by Shift+right-click on desktop and long-press on mobile.

The component must:
1. Import `isOff`, `isMobile`, `getLevel`, `onAtmoChange` from `../../lib/atmosphere`
   - Also need a way to set intensity. Import `window.__atmo__` directly for the `set` call, OR add a `setLevel(level)` export to `src/lib/atmosphere.ts` that calls `window.__atmo__?.set({ atmo: level })`.
   - **Decision**: Add a `setLevel(level: 'off' | 'subtle' | 'full')` export to `src/lib/atmosphere.ts` in this task. This keeps the API clean for the intensity selector in the menu.

2. Render the menu HTML (hidden by default):
   ```html
   <nav class="atmo-context-menu" id="context-menu" role="menu" aria-label="快捷菜单" hidden>
     <button role="menuitem" data-action="terminal">🖥️ 秘密终端</button>
     <button role="menuitem" data-action="home">🏠 首页</button>
     <button role="menuitem" data-action="sponsor">💖 赞赏</button>
     <hr />
     <div class="ctx-intensity" role="group" aria-label="氛围强度">
       <span class="ctx-intensity-label">✨ 氛围</span>
       <button role="menuitem" data-action="atmo-off" data-level="off">💤 关</button>
       <button role="menuitem" data-action="atmo-subtle" data-level="subtle">🌸 柔</button>
       <button role="menuitem" data-action="atmo-full" data-level="full">✨ 满</button>
     </div>
   </nav>
   ```

3. Style the menu:
   - `.atmo-context-menu`: `position: fixed; z-index: 10000; background: var(--color-bg-elevated); border: 1px solid var(--color-border); border-radius: 12px; padding: 0.5rem 0; min-width: 180px; box-shadow: 0 8px 32px rgba(0,0,0,0.15); backdrop-filter: blur(10px);`
   - Menu items: `display: block; width: 100%; padding: 0.5rem 1rem; border: none; background: none; cursor: pointer; font-size: 0.9rem; color: var(--color-fg); text-align: left; font-family: inherit;`
   - Menu item hover: `background: var(--color-surface);`
   - `.ctx-intensity`: `display: flex; align-items: center; gap: 0.3rem; padding: 0.3rem 1rem;`
   - `.ctx-intensity-label`: `font-size: 0.85rem; color: var(--color-fg-muted); margin-right: 0.3rem;`
   - Highlight the current intensity level button with `background: var(--color-accent-subtle); border-radius: 6px;`
   - Menu entrance animation: `opacity: 0 → 1, scale(0.95) → scale(1)` over 150ms
   - All scoped inside `<style>` (Astro scoped styles)

4. In the `<script>` block:
   - Guard: if `isOff()`, return early (no context menu when atmosphere is off)
   
   **Desktop (Shift+right-click):**
   - Listen for `contextmenu` event on `document`
   - Check `event.shiftKey` — if NOT pressed, return (let browser handle default contextmenu). Per D-04 and ATM-03: "Shift+right-click only — never hijack global contextmenu"
   - If Shift is pressed: `event.preventDefault()`, show the menu at `event.clientX, event.clientY`
   - Position the menu so it doesn't overflow the viewport (flip horizontally/vertically if near edge)
   
   **Mobile (long-press):**
   - If `isMobile()`, listen for `touchstart` with a 500ms timer
   - On `touchend` or `touchmove` (beyond 10px), clear the timer
   - If timer fires: show the menu at the touch position
   - Prevent the browser's default long-press context menu

   **Menu actions:**
   - `terminal`: dispatch a custom event `open-secret-terminal` on `document` (the EasterEggIsland listens for this)
   - `home`: `window.location.href = '/'`
   - `sponsor`: show a tooltip/alert with "赞赏功能即将上线~" (placeholder)
   - `atmo-off/subtle/full`: call `window.__atmo__.set({ atmo: level })`, update the highlighted button
   
   **Dismiss:**
   - Click anywhere outside the menu → hide it
   - Press Escape → hide it
   - After any menu item action → hide it
   
   - Subscribe to `onAtmoChange`: update the highlighted intensity button when the level changes externally (e.g., user clicks the header IntensityBadge)

5. Update `src/lib/atmosphere.ts` to export `setLevel`:
   ```ts
   export function setLevel(level: 'off' | 'subtle' | 'full'): void {
     window.__atmo__?.set({ atmo: level });
   }
   ```
   Add this export to the existing atmosphere.ts file.

3. In the `<script>` block:

   **Konami code detection (per D-05, ATM-06, Pitfall 5):**
   ```js
   const KONAMI_CODE = [
     'ArrowUp', 'ArrowUp', 'ArrowDown', 'ArrowDown',
     'ArrowLeft', 'ArrowRight', 'ArrowLeft', 'ArrowRight',
     'KeyB', 'KeyA'
   ];
   let konamiIndex = 0;
   ```
   - Listen for `keydown` on `document`
   - **Input guard (critical — per ATM-06 and Pitfall 5):** Check `event.target.tagName` — if `INPUT`, `TEXTAREA`, or `event.target.isContentEditable`, return immediately. This prevents "BA" typed in a textarea from triggering the easter egg.
   - Match `event.code` against `KONAMI_CODE[konamiIndex]`
   - On match: increment index. If index reaches 10 (full sequence): reset index, call `openTerminal()`
   - On mismatch: reset index to 0
   
   **Also listen for custom event from ContextMenuIsland:**
   - `document.addEventListener('open-secret-terminal', openTerminal)`

   **Terminal commands:**
   The `openTerminal()` function:
   - Unhide the terminal element
   - Focus the input
   - Print welcome message: "🌸 欢迎来到秘密终端！输入 help 查看可用命令。"
   
   The terminal processes these commands:
   - `help` → list all commands
   - `whoami` → "merlinalex — 二次元の住人"
   - `secret` → "你发现了隐藏彩蛋！🎉 这个站点是用 Astro + Tailwind CSS 构建的二次元个人站。"
   - `theme` → show current theme: "当前主题：{light/dark}"
   - `atmo` → show current intensity: "氛围强度：{off/subtle/full}"
   - `date` → show current date with a fun format: "📅 2026年6月4日 — 今天也是美好的一天~"
   - `neko` → print ASCII art cat: a small kawaii cat face in ASCII
   - `clear` → clear terminal output
   - `exit` / `quit` → close the terminal
   - Any unknown command → "🤔 未知命令。输入 help 查看可用命令。"

   Input handling:
   - Listen for `keydown` on the terminal input with `Enter` to submit
   - On submit: read input value, print `❯ {command}` as user line, execute command, print result as system line, clear input
   - Close button: hide the terminal, refocus document

   **Escape key:** When terminal is open, pressing Escape closes it.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keyboard events → easter egg | Global keydown listener; guard checks event.target.tagName |
| context menu actions → navigation | Menu items have hardcoded actions, no user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-05-10 | Information Disclosure | Konami code keydown listener | accept | Listener only processes arrow keys + B/A codes. No data is collected or exfiltrated. Input guard prevents capture from form fields. |
| T-05-11 | Tampering | Context menu actions | accept | All actions are hardcoded (navigate to /, dispatch custom event, call __atmo__.set). No user input flows into action handlers. |
| T-05-12 | Elevation of Privilege | Terminal commands | accept | Terminal is purely cosmetic — commands only read state (theme, atmo level, date) and display text. No file system access, no network requests, no state mutation except closing the terminal. |
</threat_model>
