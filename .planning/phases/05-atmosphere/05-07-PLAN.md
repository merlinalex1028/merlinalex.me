<task type="auto">
  <name>Task 1: Fix BGM cross-page persistence</name>
  <files>src/components/atmosphere/BGMIsland.astro</files>
  <action>
Fix `src/components/atmosphere/BGMIsland.astro`. The BGM stops playing after page navigation because Astro View Transitions destroy the APlayer JavaScript instance during the DOM swap, even though `transition:persist` preserves the DOM element.

Root cause: `transition:persist` only preserves the DOM element — it does NOT preserve JavaScript state (APlayer instance, audio element, event handlers). When the page swaps, the script module context is destroyed and the APlayer audio element loses its playback state.

Fix strategy: Use View Transition lifecycle hooks to save and restore BGM state.

Changes required:

1. **Keep `transition:persist="bgm-player"` on the root div** — this ensures the DOM element survives navigation.

2. **Save playback state before swap**: Listen for `astro:before-swap` event. Save the current audio state to localStorage:
   ```
   document.addEventListener('astro:before-swap', () => {
     const audio = document.querySelector('#aplayer-container audio') as HTMLAudioElement | null;
     if (audio) {
       localStorage.setItem('bgm:time', String(audio.currentTime));
       localStorage.setItem('bgm:playing', String(!audio.paused));
     }
   });
   ```

3. **Restore playback state after swap**: Listen for `astro:after-swap` event. After the swap, the persisted element is in the new DOM but the APlayer instance is gone. Check if `localStorage.getItem('bgm:unmuted') === 'true'` and re-initialize APlayer:
   ```
   document.addEventListener('astro:after-swap', () => {
     if (localStorage.getItem('bgm:unmuted') === 'true') {
       // Reset the loaded flag so loadAPlayer() can run again
       aplayerLoaded = false;
       // Hide unmute button (already unmuted)
       hideUnmute();
       // Re-load APlayer
       void loadAPlayer().then(() => {
         // Restore playback position after APlayer loads
         const savedTime = parseFloat(localStorage.getItem('bgm:time') || '0');
         const wasPlaying = localStorage.getItem('bgm:playing') === 'true';
         if (savedTime > 0 || wasPlaying) {
           // Wait for APlayer to initialize its audio element
           setTimeout(() => {
             const audio = document.querySelector('#aplayer-container audio') as HTMLAudioElement | null;
             if (audio && savedTime > 0) {
               audio.currentTime = savedTime;
             }
             if (audio && wasPlaying) {
               audio.play().catch(() => {});
             }
           }, 500);
         }
       });
     }
   });
   ```

4. **Handle the `aplayerLoaded` flag**: The variable `aplayerLoaded` is declared in the module scope. When the script re-runs after navigation (Astro re-evaluates scripts for new pages), `aplayerLoaded` resets to `false`. But since the script runs fresh, `loadAPlayer()` should work. The issue is that the script's `if (isOff())` / `else if (localStorage.getItem('bgm:unmuted'))` block runs on initial script execution. If the script runs AFTER the astro:after-swap handler, there could be a double-init. To prevent this:
   - Move the `astro:after-swap` listener to be registered BEFORE the initial gate logic
   - Add a guard in `loadAPlayer()`: check if a `<meting-js>` element already exists in the container before creating a new one

5. **Ensure the unmute button works after navigation**: The `unmuteBtn?.addEventListener('click', handleUnmute)` line runs when the script initializes. Since `transition:persist` preserves the button element, and the script re-runs on each page, the click handler is re-attached. This should work correctly.

6. **Test the fix**: After changes, verify by:
   - Start BGM on home page (click unmute)
   - Navigate to /articles via client-side link
   - BGM should continue playing without restart
   - Navigate back to / — BGM should still be playing

<task type="auto">
  <name>Task 2: Fix ContextMenuIsland — replace hidden attribute with CSS visibility</name>
  <files>src/components/atmosphere/ContextMenuIsland.astro</files>
  <action>
Fix `src/components/atmosphere/ContextMenuIsland.astro`. The context menu is not visible during UAT because the HTML `hidden` attribute prevents CSS transitions from working. When `menu.hidden = false` is called, the element jumps from `display: none` to its default display without any opacity/transform transition.

Root cause: The `hidden` attribute sets `display: none !important`. CSS transitions cannot animate from `display: none` — the browser skips the transition entirely. The menu's CSS uses `opacity: 0 → 1` and `transform: scale(0.95) → scale(1)` transitions, but these never play because the element goes from `display: none` directly to `display: block` (skipping the transition).

Fix strategy: Replace the `hidden` attribute with CSS-based visibility control using a class.

Changes required:

1. **Remove `hidden` from the HTML**: Change the nav element:
   ```
   Before: <nav class="atmo-context-menu" id="context-menu" role="menu" aria-label="快捷菜单" hidden>
   After:  <nav class="atmo-context-menu atmo-context-menu--hidden" id="context-menu" role="menu" aria-label="快捷菜单">
   ```

2. **Update the CSS** to use the hidden class instead of the hidden attribute:
   ```
   .atmo-context-menu--hidden {
     opacity: 0;
     transform: scale(0.95);
     pointer-events: none;
     visibility: hidden;
   }

   .atmo-context-menu.ctx-visible {
     opacity: 1;
     transform: scale(1);
     pointer-events: auto;
     visibility: visible;
   }
   ```
   Keep the existing transition properties on `.atmo-context-menu`.

3. **Update showMenu() function**: Replace `menu.hidden = false` with class manipulation:
   ```
   function showMenu(x: number, y: number): void {
     if (!menu) return;
     // ... positioning code ...
     menu.classList.remove('atmo-context-menu--hidden');
     menu.classList.add('ctx-visible');
     visible = true;
     highlightLevel(getLevel());
   }
   ```

4. **Update hideMenu() function**: Replace `menu.hidden = true` with class manipulation:
   ```
   function hideMenu(): void {
     if (!menu) return;
     menu.classList.remove('ctx-visible');
     menu.classList.add('atmo-context-menu--hidden');
     visible = false;
   }
   ```

5. **Verify the event listener attachment**: The script currently has `if (!isOff() && menu)` guard. If `isOff()` returns true, the contextmenu listener is never attached. Add the same `waitForAtmo` pattern used in CursorTrailIsland to handle the race condition where `window.__atmo__` might not be set yet:
   ```
   function initContextMenu(): void {
     if (isOff() || !menu) return;
     // ... all existing event listener code ...
   }

   if ((window as any).__atmo__) {
     initContextMenu();
   } else {
     const check = setInterval(() => {
       if ((window as any).__atmo__) {
         clearInterval(check);
         initContextMenu();
       }
     }, 50);
     setTimeout(() => clearInterval(check), 2000);
   }
   ```

6. **Keep the `hidden` attribute approach for the terminal in EasterEggIsland** — that component works (Konami code passes UAT), so don't change it.
