  <name>Task 2: Create reusable JsonLd component and wire into article pages</name>
  <files>src/components/seo/JsonLd.astro, src/pages/articles/[id].astro</files>

Create a reusable `JsonLd.astro` component and wire it into the article detail page. Per D-02, this covers Article + Person + BreadcrumbList schemas for articles only.

**Create `src/components/seo/JsonLd.astro`:**
A minimal Astro component that accepts one or more JSON-LD schema objects and renders them as `<script type="application/ld+json">` tags in the page.

Props interface (TypeScript via Astro props):
- `schemas: Record<string, unknown>[]` — array of JSON-LD schema objects to render

The component renders one `<script type="application/ld+json">` tag per schema object, with `set:html={JSON.stringify(schema)}` for safe HTML injection. Each script tag should have the `is:inline` attribute so it's rendered as-is without Astro processing.

Example usage:
```
<JsonLd schemas={[articleSchema, personSchema, breadcrumbSchema]} />
```

**Wire into `src/pages/articles/[id].astro`:**
Import `JsonLd` from `'../../components/seo/JsonLd.astro'` and add it inside the `<BaseLayout>` component, before the article content.

Build three schema objects in the frontmatter script:

1. **Article schema** (per D-02):
```typescript
const articleSchema = {
  '@context': 'https://schema.org',
  '@type': 'Article',
  headline: article.data.title,
  description: article.data.description ?? article.data.title,
  author: {
    '@type': 'Person',
    name: 'merlinalex',
    url: Astro.site?.toString(),
  },
  datePublished: article.data.publishedAt.toISOString(),
  dateModified: (article.data.updatedAt ?? article.data.publishedAt).toISOString(),
  image: article.data.cover ? new URL(article.data.cover, Astro.site).toString() : undefined,
  url: permalink,
  mainEntityOfPage: {
    '@type': 'WebPage',
    '@id': permalink,
  },
};
```

Filter out `undefined` fields from the article schema (especially `image` when no cover exists) to avoid empty values in the JSON-LD output.

2. **Person schema** (per D-02 — basic profile):
```typescript
const personSchema = {
  '@context': 'https://schema.org',
  '@type': 'Person',
  name: 'merlinalex',
  url: Astro.site?.toString(),
  image: new URL('/avatar.png', Astro.site).toString(),
  sameAs: [
    'https://github.com/merlinalex',
    // Add Twitter if available, otherwise omit
  ],
};
```

Only include `sameAs` entries that actually exist. If there's no Twitter/social link, omit the `sameAs` field entirely rather than passing an empty array.

3. **BreadcrumbList schema** (per D-02 — 3-level: Home > Articles > Title):
```typescript
const breadcrumbSchema = {
  '@context': 'https://schema.org',
  '@type': 'BreadcrumbList',
  itemListElement: [
    {
      '@type': 'ListItem',
      position: 1,
      name: '首页',
      item: Astro.site?.toString(),
    },
    {
      '@type': 'ListItem',
      position: 2,
      name: '文章',
      item: new URL('/articles/', Astro.site).toString(),
    },
    {
      '@type': 'ListItem',
      position: 3,
      name: article.data.title,
      item: permalink,
    },
  ],
};
```

Render all three schemas via `<JsonLd schemas={[articleSchema, personSchema, breadcrumbSchema]} />` inside `<BaseLayout>`, placed after the opening `<BaseLayout>` tag and before the article layout div.

**BaseLayout head injection:**
The `<JsonLd>` component renders `<script>` tags. In Astro, scripts placed inside a component that's rendered in the `<body>` will appear in the `<body>`, not `<head>`. For JSON-LD, this is acceptable — Google's documentation states JSON-LD can be placed anywhere in the document (head or body). Do NOT add a `<slot name="head">` to BaseLayout just for this — it's unnecessary complexity. The `<script type="application/ld+json">` tags in the body work fine.
